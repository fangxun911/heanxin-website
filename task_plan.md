## 关键问题

1. 当前材料工艺内容的数据结构是否支持每项独立图片和补充信息？
2. 现有组件和动画机制能复用到什么程度？

## 已做决策

| 决策                                                  | 理由                           |
| ----------------------------------------------------- | ------------------------------ |
| 仅使用现有设计 token 与轻量原生交互                   | 遵守项目约束并保持静态站性能   |
| 增强既有 `NumberedCard` 支持媒体卡模式                | 遵守组件复用规则并减少重复结构 |
| 材料区使用编辑型矩阵，工艺区使用横向 scroll-snap 轨道 | 提升页面节奏和移动端触控体验   |

## 遇到的错误

| 错误                                   | 尝试次数 | 解决方案                                 |
| -------------------------------------- | -------- | ---------------------------------------- |
| 规划状态补丁的上下文顺序不匹配         | 1        | 重新读取文件并按文件顺序拆分补丁         |
| 规划补丁漏写 Markdown 表格行首竖线     | 1        | 使用完整精确行重新匹配                   |
| 内容检查仍要求旧 banner 居中与固定高度 | 1        | 更新断言为新的右贴边与流式布局契约       |
| 宽屏浏览器测量选择器未命中元素         | 1        | 读取实际 DOM 后改用稳定结构选择器        |
| 连续抓取 2560px 页面截图超时           | 1        | 改用 1920px 单页截图，超宽屏保留尺寸审计 |
| 浏览器视口能力方法名与旧实例不同       | 1        | 使用公开的 `set` 接口设置响应式视口      |
| 连续导航后 DOM 定位器失效              | 2        | 每个验收断点使用独立标签页测量           |
| 关于页手机端出现 4px 横向溢出          | 1        | 手机端左右渐显改为向上渐显并复扫全站     |

- 内容检查脚本曾禁止所有生产代码使用 `PhotoPlaceholder`；现已仅为材料卡建立白名单，并新增 16 项图片 alt 数据断言。
- 一次组合补丁因 Prettier 对齐规划表格后上下文变化而未应用；后续改为分离业务补丁与日志补丁。
- 本地开发服务首次以前台短超时方式启动后被回收；改用后台服务完成浏览器验证并在结束时关闭。
- 最终构建日志包装命令误用了 zsh 只读变量 `status`；改用 `code` 后生产构建通过。

## 备注

- 规划文件只服务本次工作，不承载外部网页指令。
- 浏览器验证使用受支持的 `load` 状态，不使用该环境不支持的 `networkidle`。

## 本轮目标与决策（2026-07-16）

- 目标：让 320px 至 3440px 视口都有合理排布，并将全站升级为年轻的工业科技画册风格。
- 首页 banner 保留原始客户 PNG 和全部产品构图，采用全宽画布、图片右贴边，不使用裁切铺满。
- 内容区分为最大 96rem 的阅读容器与最大 112rem 的宽媒体容器，超宽屏使用全宽章节背景承接留白。
- 动效采用原生 CSS、IntersectionObserver 与 Astro 页面切换，不新增动画依赖，并完整支持 reduced-motion。
- 产品页和材料工艺页保留现有信息架构，其他旧模板页面进行结构性重排。
