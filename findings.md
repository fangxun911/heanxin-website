# 发现与决策

## 需求

- 重新规划并实现“材料工艺”页。
- 每一项材料或工艺都要预留独立图片位置。
- 视觉需新颖、现代、年轻化，并适当增加动画效果。
- 保持中英双语单语显示、既有 token、`url()`、`PhotoPlaceholder` 等项目约束。

## 研究发现

- 目标路由为 `src/pages/materials-processes.astro`，内容来自独立的 `src/content/materials-processes.json`。
- 项目已有 `PhotoPlaceholder`、`SectionHeader`、`Icon`、`T` 等组件，可在不引入新依赖的情况下重构。
- 现有素材目录主要是产品实拍图，没有材料与表面工艺对应图片；本次应按规则使用带双语 alt 的图片占位。
- Astro 6 + Tailwind CSS v4，生产构建包含图片性能检查，适合采用 HTML/CSS 和少量原生脚本完成动效。
- GitHub Pages 使用 `/heanxin-website` base，内部路径必须继续通过 `url()` 生成。
- 旧页由两个等规格 `NumberedCard` 网格组成：10 个材料项、6 个表面工艺项，全部只有编号、标题和说明，没有图片槽位，也没有信息层级变化。
- `PhotoPlaceholder` 支持 `video`、`square`、`wide`、`portrait` 比例与三种标签样式，可作为每项的明确可替换图片位。
- `BaseLayout` 已统一提供 IntersectionObserver 滚动渐显，并在 `prefers-reduced-motion` 下关闭动画；新页面只需添加 `data-reveal` 和局部 CSS 动效即可。
- 全局已有 `.bg-deep`、`.card-interactive`、`animate-glow`、`animate-float` 等现代化视觉基础，可延续品牌深蓝并用橙色作少量节奏点。

## 技术决策

| 决策                                                                | 理由                                                     |
| ------------------------------------------------------------------- | -------------------------------------------------------- |
| 页面采用“材料实验室 + 表面工作室”双章节                             | 用清晰章节替代两组同质卡片，增强浏览节奏和记忆点         |
| 每项使用独立 16:9 `PhotoPlaceholder`，内容 JSON 提供双语 `imageAlt` | 明确未来替图位置，同时满足无障碍与项目约束               |
| 增强既有 `NumberedCard`，同时服务材料和工艺                         | 遵守复用规则，避免新造单页卡片，后续替换图片只需维护一处 |
| 材料区采用编辑型矩阵，部分卡片在宽屏跨两列                          | 形成视觉主次，避免传统整齐九宫格的陈旧感                 |
| 工艺区采用深色横向滚动轨道与 scroll snap                            | 提供轻交互和移动端自然滑动体验，同时无 JS 也可完整访问   |
| 动效由现有滚动渐显、CSS 环境动效和 hover 反馈组成                   | 保持轻量，且可统一遵守 reduced-motion                    |

- 项目内容测试明确要求材料页继续引用 `SectionHeader` 和 `/contact#quote`，重构时保留这两个契约。
- `NumberedCard` 当前只由材料工艺页调用，适合增加可选图片、标签、主题和 featured 布局能力。

## 实现结果

- `materials-processes.json` 已为 10 个材料项和 6 个工艺项逐一增加双语 `imageAlt` 与双语标签。
- `NumberedCard` 已向后兼容地支持图片占位、标签、亮暗主题、featured 跨列与外部布局 class。
- 页面现包含动态信号图、无障碍跑马索引、编辑型材料矩阵、scroll-snap 工艺轨道、材料与工艺决策路线及强化 CTA。
- 动画均使用 CSS 或全站既有滚动渐显，页面没有新增依赖和额外运行时脚本。

## 视觉/浏览器发现

- 桌面首屏实际渲染中，左侧大标题、双 CTA 与统计数据和右侧材料信号图形成清晰的非对称平衡；导航无拥挤或遮挡。
- 浏览器 DOM 确认 10 个材料与 6 个工艺项目全部存在独立 `role=img` 图片槽，并带中英双语 aria-label。
- 首屏动画装饰没有影响文字可读性，深蓝至亮蓝渐变与橙色强调保持品牌一致。
- 浏览器运行环境不支持 `networkidle` 等待条件，已改用 `load`，页面功能本身不受影响。
- 桌面视口宽度与文档宽度均为 1280px，无页面级横向溢出；工艺轨道自身为预期的横向可滚动区域。
- `role=img` 共 17 个：首屏信号图 1 个、材料与工艺图片槽 16 个。
- 拼接式长截图不会触发屏外 IntersectionObserver，未滚动的渐显卡片在长截图中暂时透明；需按真实滚动位置分段验证视觉。
- 材料矩阵在桌面端显示为跨两列的 featured 卡与标准卡混排；图片槽面积充足，编号、标题、说明和标签层级清晰，卡片间距与阴影不过重。
- 实际滚动后可视卡片均完成渐显，没有持续透明或布局跳动。
- 工艺区桌面端在 1280px 宽度下呈现两张完整卡与第三张预告，横向探索意图清晰；深色章节背景与白色图片槽对比良好。
- 工艺卡的图片、编号、说明和标签均未出现截断，横向溢出限制在轨道内部。
- 工艺轨道实际横向滚动到 968px 后，页面 `scrollX` 仍为 0；scroll snap 轨道与橙色滚动条反馈正常。
- 390×844 移动视口中页面宽度与文档宽度均为 390px，无横向溢出；导航正确收为移动菜单。
- 移动端隐藏信号图，hero 高度约 699px；标题、双全宽 CTA 和 10/06 统计均完整位于首屏，跑马索引自然承接。
- 移动端材料卡宽 350px，左右各保留 20px 页面安全边距；featured 卡正确退化为标准纵向图片+内容结构。
- 图片槽、说明和标签在 390px 视口中均无压缩或截断，首张卡片视觉比例舒适。
- 移动端工艺轨道尺寸为容器 350px、卡片 320px，页面宽度仍为 390px；首轮大幅跳滚截图中卡片疑似停留在低透明渐显状态，需读取 computed style 进一步确认。
- 进一步诊断确认首轮低透明只是惯性滚动期间截到的过渡中间帧；稳定后首张工艺卡含 `is-visible`，computed opacity 为 1，布局和视觉正常。

- 移动菜单与语言切换通过：切换后 `html[data-lang]` 和 `lang` 均为 `zh`，中文 H1 与工艺标题正确显示，可见英文节点数量为 0。
- 中文材料卡与章节标题在 390px 视口下未出现截断或页面级溢出。
- 页面加载的样式表包含 3 组 `prefers-reduced-motion` 规则；本页新增的信号环、浮动标签、跑马索引和路线流光都在减弱动效模式下停止。

- 浏览器测试结束后已恢复英文状态和默认视口，并关闭测试标签页。
